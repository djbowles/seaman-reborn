## Codebase Patterns
- Build backend is `setuptools.build_meta` (NOT `setuptools.backends._legacy`)
- pytest-asyncio uses `asyncio_mode = "auto"` in pyproject.toml
- All stub files have docstrings with "Stub - implementation in US-XXX"
- Package installed in editable mode: `pip install -e ".[dev]"`
- Quality gates: `ruff check src/ tests/` + `python -m pytest tests/ -x --tb=short` + `python -m seaman_brain --version`
- Use `datetime.UTC` not `timezone.utc` (ruff UP017 rule)
- Run ruff via `python -m ruff` (not bare `ruff`) — bare command not on PATH
- Cloud LLM providers (openai, anthropic) are in `[cloud]` optional deps: `pip install -e ".[cloud]"`
- Anthropic API separates system messages from conversation — pass via `system` kwarg, not in `messages` array
- LanceDB async: `connect_async()`, `table.search()` is a coroutine (await it), then chain `.limit(k).to_list()`
- LanceDB `list_tables()` returns a response object — access `.tables` for the list of names (replaces deprecated `table_names()`)
- LanceDB stores timestamps as ISO strings, reconstruct with `datetime.fromisoformat()`
- prompt_toolkit PromptSession must be lazy-initialized (not in __init__) — fails in non-console environments like pytest
- For local-import modules (pyttsx3, etc.), mock via `patch.dict(sys.modules, {"pkg": mock})` not `patch("module.pkg")`
- pyttsx3 engines are NOT thread-safe — use single-thread ThreadPoolExecutor; create fresh engine per operation
- Pygame mocked in tests via `patch.dict(sys.modules, {"pygame": mock})` + force reimport after mocking; mock needs int constants (QUIT=256, KEYDOWN=768, K_ESCAPE=27)
- For modules that import numpy (via types.py): use module-level `sys.modules["pygame"] = mock` + import once, NOT per-test `patch.dict` (which removes numpy between tests, causing C extension reload error)
- Background asyncio for GUI: `asyncio.new_event_loop()` + `loop.run_forever()` + daemon thread; `run_coroutine_threadsafe()` to bridge sync->async
- For gui test modules with shared pygame mock: re-install `sys.modules["pygame"]` in autouse fixture AND patch `module.pygame` on the imported module directly (e.g. `hud_mod.pygame = _pygame_mock`) — other test_gui modules overwrite the mock at collection time, and the target module may have captured a different mock's reference
- MoodEngine.calculate_mood() requires a non-None TraitProfile — use `TraitProfile()` as default when ConversationManager traits aren't available
