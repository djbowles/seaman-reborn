### Key Design Decisions

**Package manager**: `pyproject.toml` with pip. No Poetry or uv dependency -- keeping it simple since the machine has pip 25.3 and Python 3.13. The pyproject.toml will use `[project.scripts]` to define `seaman-brain = "seaman_brain.__main__:main"` as a console entry point.

**Configuration format**: TOML. Python 3.11+ has `tomllib` in the standard library. Stage-specific personality overrides live in separate TOML files under `config/stages/` and are merged with the default config at runtime.

**Validation**: Pydantic v2 for config validation and shared types. This gives type safety and easy serialization without heavy boilerplate.

**Testing**: pytest with no heavy test framework. Tests use mocks for LLM calls (no real inference in unit tests). Integration tests optionally hit a real Ollama instance when available.

### 3.2 LLM Provider Abstraction (`llm/`)

**Protocol** (`base.py`):
```python
from typing import Protocol, AsyncIterator

class LLMProvider(Protocol):
    async def chat(
        self,
        messages: list[dict[str, str]],
        temperature: float = 0.8,
        max_tokens: int = 512,
    ) -> str:
        """Send messages, get complete response."""
        ...

    async def chat_stream(
        self,
        messages: list[dict[str, str]],
        temperature: float = 0.8,
        max_tokens: int = 512,
    ) -> AsyncIterator[str]:
        """Stream response token by token."""
        ...
```

**Ollama Provider** (`ollama_provider.py`):
```python
from ollama import AsyncClient

class OllamaProvider:
    def __init__(self, model: str, base_url: str):
        self.model = model
        self.client = AsyncClient(host=base_url)

    async def chat(self, messages, temperature=0.8, max_tokens=512) -> str:
        response = await self.client.chat(
            model=self.model,
            messages=messages,
            options={"temperature": temperature, "num_predict": max_tokens},
        )
        return response["message"]["content"]

    async def chat_stream(self, messages, temperature=0.8, max_tokens=512):
        async for chunk in await self.client.chat(
            model=self.model,
            messages=messages,
            options={"temperature": temperature, "num_predict": max_tokens},
            stream=True,
        ):
            yield chunk["message"]["content"]
```

**OpenAI Provider** (`openai_provider.py`): Uses `openai.AsyncOpenAI` client with the same `chat` / `chat_stream` interface, translating to `client.chat.completions.create()`.

**Anthropic Provider** (`anthropic_provider.py`): Uses `anthropic.AsyncAnthropic` client, translating to `client.messages.create()`. Note: Anthropic uses a `system` parameter rather than a system message in the messages list, so the provider must extract the system message and pass it separately.

**Factory** (`factory.py`):
```python
def create_provider(config: LLMConfig) -> LLMProvider:

        case "openai":
            return OpenAIProvider(config.model, config.openai_api_key)
        case "anthropic":
            return AnthropicProvider(config.model, config.anthropic_api_key)

**Prompt Builder** (`prompt_builder.py`):

This is the most critical module. It assembles the full system prompt from:
1. Core identity text (who the creature is, its backstory)
2. Current stage personality traits (translated to behavioral instructions)
3. Negative constraints (forbidden phrases and patterns)
4. Retrieved semantic memories (injected as "things you know about this human")
5. Creature state context (mood, relationship level, hunger)
6. Environmental context (future: webcam emotion data, time of day)

The prompt builder outputs a single system message string. Example structure:

```
You are an ancient creature known as a Seaman, currently in your {stage} form.

- Never use phrases like: {forbidden_phrases}

- Never apologize for being rude
- Never break character
- Keep responses under {max_words} words
- You MUST ask at least one probing personal question per response

THINGS YOU REMEMBER ABOUT THIS HUMAN:
{retrieved_semantic_memories}

### 3.4 Memory System (`memory/`)

**Episodic Memory** (`episodic.py`):
```python
from collections import deque
from dataclasses import dataclass, field
from datetime import datetime

@dataclass
class Message:
    role: str          # "user" or "assistant"
    content: str
    timestamp: datetime = field(default_factory=datetime.now)

class EpisodicMemory:
    def __init__(self, max_size: int = 20):
        self._buffer: deque[Message] = deque(maxlen=max_size)

    def add(self, role: str, content: str) -> None:
        self._buffer.append(Message(role=role, content=content))

    def get_messages(self) -> list[dict[str, str]]:
        """Return messages formatted for LLM context."""
        return [{"role": m.role, "content": m.content} for m in self._buffer]

    def get_recent_text(self, n: int = 5) -> str:
        """Return last N messages as plain text for extraction."""
        recent = list(self._buffer)[-n:]
        return "\n".join(f"{m.role}: {m.content}" for m in recent)

    def clear(self) -> None:
        self._buffer.clear()
```

**Semantic Memory** (`semantic.py`):
```python
import lancedb
from datetime import datetime

class SemanticMemory:
    def __init__(self, db_path: str, table_name: str = "memories"):
        self.db = lancedb.connect(db_path)
        self.table_name = table_name
        self._ensure_table()

    def _ensure_table(self):
        if self.table_name not in self.db.table_names():
            # Create with schema on first memory
            self._table = None
        else:
            self._table = self.db.open_table(self.table_name)

    def store(self, text: str, vector: list[float], metadata: dict) -> None:
        """Store a fact with its embedding vector and metadata."""
        record = {
            "text": text,
            "vector": vector,
            "timestamp": datetime.now().isoformat(),
            "category": metadata.get("category", "general"),
            "emotional_context": metadata.get("emotional_context", "neutral"),
        }
        if self._table is None:
            self._table = self.db.create_table(self.table_name, data=[record])
        else:
            self._table.add([record])

    def search(self, query_vector: list[float], top_k: int = 5) -> list[dict]:
        """Semantic similarity search."""
        if self._table is None:
            return []
        results = self._table.search(query_vector).limit(top_k).to_list()
        return results
```

**Embeddings** (`embeddings.py`):
```python
class EmbeddingProvider(Protocol):
    async def embed(self, text: str) -> list[float]: ...
    async def embed_batch(self, texts: list[str]) -> list[list[float]]: ...

class OllamaEmbeddings:
    def __init__(self, model: str = "all-minilm:l6-v2", base_url: str = "http://localhost:11434"):
        self.model = model
        self.client = AsyncClient(host=base_url)

    async def embed(self, text: str) -> list[float]:
        response = await self.client.embed(model=self.model, input=text)
        return response["embeddings"][0]
```

This is important: by using Ollama's `embed` endpoint with `all-minilm:l6-v2`, we avoid pulling in the heavy `sentence-transformers` and `torch` dependencies. The Ollama embedding model is only ~46 MB and runs on CPU alongside the main LLM on GPU. If the user later wants `sentence-transformers` as an alternative, the `EmbeddingProvider` protocol makes that swap trivial.

**Memory Extractor** (`extractor.py`):

Uses the same LLM (or a smaller/faster one) to extract permanent facts from recent conversation. This runs as a background coroutine, triggered every N messages or at session end.

```python
EXTRACTION_PROMPT = """Analyze this conversation excerpt and extract permanent, 
personally relevant facts about the human. Return ONLY a JSON array of facts.
Each fact should be a single declarative sentence.
Only extract facts that would be worth remembering months from now.
Do NOT include temporary states, greetings, or conversational filler.

Conversation:
{conversation_text}

Return format: ["fact 1", "fact 2", ...]"""

class MemoryExtractor:
    def __init__(self, llm: LLMProvider, embeddings: EmbeddingProvider, 
                 semantic_memory: SemanticMemory):
        self.llm = llm
        self.embeddings = embeddings
        self.semantic_memory = semantic_memory

    async def extract_and_store(self, conversation_text: str) -> list[str]:
        """Extract facts from conversation and store in semantic memory."""
        response = await self.llm.chat([
            {"role": "system", "content": "You are a fact extraction engine."},
            {"role": "user", "content": EXTRACTION_PROMPT.format(
                conversation_text=conversation_text
            )},
        ])
        facts = json.loads(response)  # parse JSON array
        for fact in facts:
            vector = await self.embeddings.embed(fact)
            self.semantic_memory.store(fact, vector, {
                "category": "extracted_fact",
                "emotional_context": "neutral",  # could be enhanced
            })
        return facts
```

**Retriever** (`retriever.py`):

Implements hybrid search combining semantic similarity with temporal recency:

```python
class HybridRetriever:
    def __init__(self, semantic_memory: SemanticMemory, 
                 embeddings: EmbeddingProvider, config: MemoryConfig):
        self.semantic_memory = semantic_memory
        self.embeddings = embeddings
        self.top_k = config.retrieval_top_k
        self.temporal_weight = config.temporal_decay_weight

    async def retrieve(self, query: str) -> list[str]:
        """Retrieve relevant memories using hybrid scoring."""
        query_vector = await self.embeddings.embed(query)
        raw_results = self.semantic_memory.search(query_vector, top_k=self.top_k * 2)
        # Re-rank with temporal decay
        scored = []
        now = datetime.now()
        for r in raw_results:
            age_hours = (now - datetime.fromisoformat(r["timestamp"])).total_seconds() / 3600
            recency_score = 1.0 / (1.0 + age_hours / 24.0)  # decays over days
            similarity_score = 1.0 - r.get("_distance", 0)   # LanceDB L2 distance
            hybrid_score = (
                (1 - self.temporal_weight) * similarity_score 
                + self.temporal_weight * recency_score
            )
            scored.append((hybrid_score, r["text"]))
        scored.sort(reverse=True)
        return [text for _, text in scored[:self.top_k]]
```

### 3.6 Conversation Manager (`conversation/manager.py`)

This is the main orchestrator. The full loop per user input:

```
1. User types message
2. Add message to episodic buffer
3. Update creature state (interaction count, mood drift)
4. Check evolution triggers
5. Retrieve semantic memories relevant to user's message
6. Build system prompt (personality + memories + creature state)
7. Assemble full message list: [system_prompt] + episodic_buffer
8. Send to LLM provider
9. Receive response
10. Post-process response (constraint filter)
11. Add response to episodic buffer
12. Update creature state (relationship, mood based on interaction)
13. Check if extraction threshold reached -> trigger background extraction
14. Return response to CLI/server
```

```python
class ConversationManager:
    def __init__(self, config: AppConfig):
        self.llm = create_provider(config.llm)
        self.episodic = EpisodicMemory(config.memory.episodic_buffer_size)
        self.embeddings = create_embeddings(config.memory)
        self.semantic = SemanticMemory(config.memory.lancedb_path)
        self.extractor = MemoryExtractor(self.llm, self.embeddings, self.semantic)
        self.retriever = HybridRetriever(self.semantic, self.embeddings, config.memory)
        self.prompt_builder = PromptBuilder(config.personality)
        self.creature = load_or_create_creature(config.creature)
        self._message_count_since_extraction = 0

    async def process_input(self, user_text: str) -> str:
        # 1-2. Buffer
        self.episodic.add("user", user_text)
        
        # 3-4. State
        self.creature.interaction_count += 1
        self.creature.total_messages += 1
        check_evolution(self.creature, self.config)
        
        # 5. Retrieve
        memories = await self.retriever.retrieve(user_text)
        
        # 6. Build prompt
        system_prompt = self.prompt_builder.build(
            stage=self.creature.stage,
            memories=memories,
            creature_state=self.creature,
        )
        
        # 7. Assemble
        messages = [{"role": "system", "content": system_prompt}]
        messages.extend(self.episodic.get_messages())
        
        # 8-9. Infer
        response = await self.llm.chat(messages)
        
        # 10. Filter
        response = apply_constraints(response, self.config.personality)
        
        # 11. Buffer response
        self.episodic.add("assistant", response)
        
        # 12. State update
        update_mood_and_relationship(self.creature, user_text, response)
        save_creature(self.creature, self.config.creature.save_path)
        
        # 13. Extraction check
        self._message_count_since_extraction += 1
        if self._message_count_since_extraction >= self.config.memory.extraction_interval_messages:
            await self.extractor.extract_and_store(
                self.episodic.get_recent_text(self._message_count_since_extraction)
            )
            self._message_count_since_extraction = 0
        
        return response
```

# Seaman Brain - Module Documentation

## Project Overview
Seaman Brain is the AI middleware for Seaman Reborn, a spiritual successor to the Dreamcast game Seaman. This is the "AI Brain Core" - a Python package that handles LLM inference, personality management, and memory persistence. It is testable in a terminal before any 3D rendering is integrated.

## Module Map

### src/seaman_brain/config.py
- Loads TOML configuration into Pydantic models
- `load_config(path)` is the entry point
- Stage-specific personality overrides are in config/stages/*.toml

### src/seaman_brain/types.py
- Shared types: EvolutionaryStage enum, Message dataclass
- Import from here, not from individual modules

### src/seaman_brain/llm/
- `base.py`: LLMProvider Protocol - the contract all providers implement
- `ollama_provider.py`: Local inference via Ollama
- `openai_provider.py`: Cloud fallback via OpenAI API (optional dependency)
- `anthropic_provider.py`: Cloud fallback via Anthropic API (optional dependency)
- `factory.py`: `create_provider(config)` returns the correct provider

### src/seaman_brain/personality/
- `traits.py`: Trait dimensions and per-stage personality profiles
- `prompt_builder.py`: Assembles the full system prompt from traits + memories + state
- `constraints.py`: Forbidden phrases and post-processing output filter

### src/seaman_brain/memory/
- `episodic.py`: Rolling deque buffer for short-term conversation history
- `semantic.py`: LanceDB wrapper for persistent vector storage
- `embeddings.py`: EmbeddingProvider protocol + OllamaEmbeddings implementation
- `extractor.py`: Background LLM-driven fact extraction from conversations
- `retriever.py`: Hybrid retrieval (semantic similarity + temporal recency)

### src/seaman_brain/creature/
- `state.py`: CreatureState dataclass (stage, mood, hunger, relationship, etc.)
- `evolution.py`: Stage transition logic with configurable thresholds
- `persistence.py`: JSON save/load for creature state

### src/seaman_brain/conversation/
- `manager.py`: Main orchestrator - wires everything together
- `context_assembler.py`: Combines system prompt + episodic buffer into LLM messages

### src/seaman_brain/cli/
- `terminal.py`: Rich terminal chat interface with debug commands
- `commands.py`: Debug command handlers (/state, /memory, /evolve, /reset)

## Testing Conventions
- Unit tests: mock all external dependencies (LLM, Ollama, filesystem)
- Integration tests: marked with `@pytest.mark.integration`, may require running Ollama
- Temp directories for LanceDB in tests: use `tmp_path` pytest fixture
- Async tests: use `pytest-asyncio` with `asyncio_mode = "auto"`

## Configuration
- Default config: config/default.toml
- Stage overrides: config/stages/{stage_name}.toml
- Environment variables: OPENAI_API_KEY, ANTHROPIC_API_KEY
- Runtime data: data/lancedb/, data/saves/ (gitignored)

### 5.4 ralph.sh

```bash
#!/usr/bin/env bash
set -e

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
TOOL="${1:---tool}"
TOOL_NAME="${2:-claude}"
MAX_ITERATIONS="${3:-25}"

# Validate tool
if [[ "$TOOL_NAME" != "claude" && "$TOOL_NAME" != "amp" ]]; then
  echo "Usage: ralph.sh --tool [claude|amp] [max_iterations]"
  exit 1
fi

# Read branch from prd.json
BRANCH=$(python -c "import json; print(json.load(open('$SCRIPT_DIR/prd.json'))['branchName'])")
echo "Ralph loop starting on branch: $BRANCH"
echo "Tool: $TOOL_NAME | Max iterations: $MAX_ITERATIONS"

# Ensure we're on the right branch
CURRENT_BRANCH=$(git branch --show-current 2>/dev/null || echo "none")
if [[ "$CURRENT_BRANCH" != "$BRANCH" ]]; then
  git checkout -b "$BRANCH" 2>/dev/null || git checkout "$BRANCH"
fi

for i in $(seq 1 "$MAX_ITERATIONS"); do
  echo ""
  echo "========================================="
  echo "Ralph iteration $i / $MAX_ITERATIONS"
  echo "========================================="
  echo "Start: $(date)" >> "$SCRIPT_DIR/progress.txt"

  if [[ "$TOOL_NAME" == "claude" ]]; then
    OUTPUT=$(claude --dangerously-skip-permissions --print < "$SCRIPT_DIR/CLAUDE.md" 2>&1)
  else
    OUTPUT=$(cat "$SCRIPT_DIR/CLAUDE.md" | amp --dangerously-allow-all 2>&1)
  fi

  echo "$OUTPUT"

  # Check for completion
  if echo "$OUTPUT" | grep -q "COMPLETE"; then
    echo ""
    echo "All stories complete! Ralph finished in $i iterations."
    exit 0
  fi

  echo "Iteration $i complete. Continuing..."
done

echo ""
echo "Max iterations ($MAX_ITERATIONS) reached. Some stories may still be incomplete."
echo "Check prd.json for remaining stories with passes: false"
exit 1
```

## 8. Risk Factors and Mitigations

**Risk**: Ollama model not pulled yet for embedding.
**Mitigation**: US-010 acceptance criteria includes a note. The CLAUDE.md gotchas section documents this.

**Risk**: LanceDB API changes between versions.
**Mitigation**: Pin `lancedb>=0.17` in pyproject.toml. The SemanticMemory wrapper abstracts the API.

**Risk**: LLM returns invalid JSON from extraction prompt.
**Mitigation**: US-013 explicitly requires graceful handling of parse failures. The extractor wraps JSON parsing in try/except.

**Risk**: System prompt too long for small models (8B with 4K context).
**Mitigation**: ContextAssembler includes token estimation and will truncate/warn if approaching limits. The default config targets 30B+ on RTX 5090 which has ample context.

**Risk**: Ralph loop commits broken code.
**Mitigation**: CLAUDE.md requires `ruff check` and `pytest` to pass before committing. Story is only marked `passes: true` after commit succeeds.
